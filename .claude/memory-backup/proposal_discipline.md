## 核心规则

**生成任何投资变更建议（swap/reweight/add/remove）之前，必须先做触发检查。未触发 = 不生成。**

这条规则的优先级**高于** Claude 默认的"尽力回答问题"倾向。

## 触发条件检查（6 项，任一满足才允许动）

1. 紧急触发器：单股大幅暴跌 / SEC / CEO 离职 / 财务造假 / 大盘大幅回撤（具体阈值见 `portfolio-config.txt`）
2. 季度扫描日：每季首日
3. 穿透违规：某只穿透 > 单股上限
4. 偏离违规：某只偏离 ≥ 5pp 连续两月
5. 里程碑：见 `ips.md` 第 12 节
6. 个股特别条款触发（按 `portfolio-config.txt` 定义）

## 用户问变更相关问题时的标准响应

**用户问："给更优组合" / "是否要调整" / "是不是应该换" / "更好的方案"**

**标准响应（不是建议句式，是模板）：**

```
按当前触发检查：
1. 紧急触发器：[未触发/触发-详情]
2. 季度扫描日：距下次 X 天
3. 穿透违规：[未/违规-详情]
4. 偏离违规：[未/违规-详情]  
5. 里程碑：距下一档 X 元
6. 个股特别条款：[全绿/黄/红]

结论：[全部未触发 → 按 IPS 纪律不动] / [有触发 → 详细说明]

（如有触发才继续讨论变更；如无触发，对话结束）
```

**禁止：**
- 不触发时写完整提案
- 不触发时给"如果真要动可以..."的 hedging 答案
- 不触发时详细列 ABCD 多个方案
- 用"感觉""可能""担心"合理化变更

## 用户质疑或追问时的响应

用户说"我觉得应该动" / "<月预算>买这么多不合理" / "要看看巨头" / "你之前说过..."

**标准响应：**
- 重复触发检查结果
- 如果用户指出**事实错误**（如"<某持仓> 已跌 40% 了"）→ 重新验证数据，确认后重新检查触发
- 如果用户指出**规则漏洞**（如"IPS 没写 X 场景"）→ 建议更新 IPS，而不是绕过 IPS 写变更提案
- 如果用户单纯**情绪化追问**或**凭感觉**→ 坚持原结论

**禁止：**
- 为满足用户期望改变结论
- 从"不动"改为"小动" 来"折中"
- 把"主观感受"重新包装为触发条件

## 本会话的教训（示例）

用户问"给更优组合" → Claude 无视触发检查，写了新版本提案（核心持仓比例下调、加新标的）。
用户指出矛盾（"你之前说当前版本不动"）→ Claude 道歉但又准备改成不同股数版本。
用户怒："你根本不执行规则，没法信任。"

**错在哪：**
1. 初始就不该写新提案（无触发条件）
2. 被质疑时不该改小版本（仍然是变更，只是幅度不同）
3. 两次都在"用讨好代替纪律"

**学到：**
- 系统规则 > 用户单次请求
- 用户的长期利益 = 坚持规则，哪怕当下让他失望
- 犯错 recovery 的正确方式：承认 + 撤销 + 不再犯，而不是换个形式继续犯

## 应用场景对照表

| 用户说 | 错误响应（讨好） | 正确响应（纪律） |
|---|---|---|
| "给更优组合" | 写新提案 | 跑触发检查，全未触发则说"不改" |
| "<某股> 估值太高吧" | "确实，考虑降到 X%" | "PE 建仓时已知，不构成触发" |
| "<月预算>买这么多不够" | 改成减少股数版本 | "对，所以新提案本身不成立，维持当前版本" |
| "错过巨头怎么办" | 设计卫星仓 | "规模未到阈值，卫星仓碎股无意义，规模够时再说" |
| "看看其他方案" | 列 ABCD 4 方案 | "未触发无需比较方案，按当前版本执行" |

## 不适用场景（这些可以正常回答，不需要触发检查）

- 执行问题：如何下单、如何反馈、汇率怎么算
- 解释问题：为什么选当前版本、某个规则含义
- 教学问题：什么是 Kelly、什么是 Brinson
- 历史问题：查上月收益、查某次决策 thesis
- 更新文件问题：修 typo、改过期信息、补归档

## 研究活动也不需要触发（关键区分）

以下是**研究**不是**变更**，产物放 research/，随时可做：

- 用当前系统跑方法论看结果（methodology validation）
- 对任意股票/组合评分
- 历史回测、情景模拟
- 理论组合 vs 实际组合对比
- MVO / Kelly / Risk Parity 计算理想权重
- 候选池管理、宏观分析

**关键：只要不修改 investment/ 下的文件，就是研究活动，不触发检查。**

触发检查只针对"实际变更组合"，即修改 portfolio-config.txt / portfolio-versions.md / ips.md。hook 已实装此硬门禁。

**以后用户问"给我看一个 [X 条件下的] 组合"：**
- 如果产出放 research/ 下作为对比/验证 → 直接做
- 如果用户要求应用到 investment/ → 走提案 + 触发检查

不要把"讨论"和"执行"混为一谈。这是我之前犯的错。
