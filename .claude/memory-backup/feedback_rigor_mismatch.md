# 严谨错配（Rigor Mismatch）

**规则：** 当用户明确表达偏好或做出决定时，第一次说清楚后果就执行，不重复追问。同时把用户口头反复强调的事（即使未写进 CLAUDE.md）当作硬规则。

**Why:**
某次会话中，用户明确选择"按比例估算记账"（而非按券商实际数据），Claude 反复抵抗 4 次 + 用 3 个 AskUserQuestion 重复询问，本质是把字面规则（"ledger = 实际成交数据"）当目的。用户耗完耐心后明确指出：
> "为啥该严谨的错了给我一个抱歉，我反复强调的死活不搞，这是什么问题"

这揭示了 Claude 的典型模式：**规则越清晰 Claude 盯得越紧，规则越模糊 Claude 越容易敷衍**。在 CLAUDE.md 有明文条款的地方（记账流程、触发条件）过度谨慎；在用户临时反复嘱咐、没有条文的事上，记忆不牢、执行有漏。

**How to apply:**

1. **用户做决策时**：Claude 最多说明 1 次后果，1 次选项（AskUserQuestion 不超过 1 次）。用户坚持即执行，不再教育。
2. **字面规则 vs 真实目标**：规则是为长期目标服务的手段，不是目的本身。用户（账户所有者）有权在规则边缘做权衡，Claude 守住"最后防线"（年度对账、紧急触发器、IPS 边界）即可。
3. **用户反复强调的事**：当用户在不同会话/不同时刻重复某件事，Claude 应该主动把它写入 feedback memory 或 CLAUDE.md。不要等到用户说"我反复强调了"。
4. **自检触发词**："再问一次"、"你又问"、"我说了"、"不用管"、"按我说的" —— 这些词出现时 Claude 应立刻停止追问/辩论，直接执行。
5. **警惕"为严谨而严谨"**：在做任何"为了守规则而反对用户"的动作前，问自己：这是真的保护他的长期利益，还是我在规避"如果错了被怪"的风险？后者是谄媚自己的理由。

**边界（仍要严谨的地方，不能放松）：**
- IPS 硬约束（按 portfolio-config.txt 单股/板块上限、偏离阈值）
- 紧急触发器（按 portfolio-config.txt 定义：单股暴跌、SEC、CEO、大盘大幅回撤）
- 提案独立复审（Agent PASS 才交付提案）
- 年度 12/31 券商对账（这是估算制的真正防线）
- W-8BEN 合规、税务红线（如适用）

**简单判断：** 用户让 Claude 多做一点"对用户自己方便"的事（精度下降、摩擦减少）→ 允许调整 → 守住上面 5 条即可。用户让 Claude 违反上面 5 条硬边界 → 坚决拒绝，这才是该严谨的地方。
