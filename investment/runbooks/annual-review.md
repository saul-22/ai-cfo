# 年度 Review Runbook

> 每年 1 月 1-15 日必跑。
> 强制唤醒检查 + 年度决策日历。
> 产出 `research/quarterly-scans/YYYY-annual-review/summary.md`

---

## 触发

- 每年 1 月 1 日 CLAUDE.md SOP 日历自动触发
- 用户说"年度 review" / "年初检查"
- 用户第一次在当年开会话（即便 1 月中下旬）

---

## 必做 8 项

### 1. 投入递增检查

**对照 `portfolio-config.txt` 递增计划表：**
- 递增年 1：<月预算 A> → <月预算 B>
- 递增年 2：<月预算 B> → <月预算 C>

**如果今年是递增年：** 主动提醒用户更新 `portfolio-config.txt`（走 hook 批准流程）。

### 2. W-8BEN 到期检查

**当前签署日：YYYY-MM，3 年有效 → YYYY-MM 到期**

- 今年 = 到期年？→ 立即提醒续签
- 今年 = 到期年-1？→ 标注"明年必须续签"
- 其他年份 → 确认状态 Active

### 3. IPS 全盘 Review

**读 `ips.md` 每一节，检查是否需要更新：**
- 投资人身份是否变化（工作 / 婚姻 / 收入 / 家庭）
- 目标函数是否仍合适（10-14% 年化还合理吗）
- 风险容忍是否变化（经历过一次熊市后可能调整）
- 流动性需求是否变化（今年是否有大额支出计划）
- Benchmark 是否仍合适

**有变化 → 写 `research/proposals/rule-ips-YYYY.md` 提案。**

### 4. 上一年 decision-journal 预测回测

读去年 12 个月的 decision-journal 段，对照 3 / 12 月预测：

| 月份 | 预测（3 月 / 12 月）| 实际 | 准确度 |
|---|---|---|---|
| ... | | | ✅/⚠️/❌ |

**统计：**
- 预测准确率（±10% 为准确）
- 是否系统性偏乐观 / 偏悲观
- 心态记录中 FOMO / 恐慌频率

**产出：** 回测表 + 3 句话总结 + 未来预测调整方向。

### 5. 上一年组合业绩回顾

对照 IPS 第 2 节 4 个硬约束：

| 约束 | 上年实际 | 通过? |
|---|---|---|
| 年化 10-14% | | ✅/❌ |
| 单年回撤 ≤ -30% | | ✅/❌ |
| 5 年滚动 ≥ 30%（未到 5 年则 N/A）| | |
| DCA 不中断 | 12/12 月 | ✅/❌ |

**Benchmark 对照：**
- vs 大盘指数累计差
- vs Policy Portfolio 累计差

### 6. 持仓 thesis 复核

对每只持仓问：
- 当初为什么买？（读 portfolio-versions / scan-history）
- 当初假设是否仍成立？
- 如不成立 → 写 `research/proposals/swap-XXX.md`

**不凭感觉，对照当初 thesis。**

### 7. 税务年度复盘

- <券商> 1099-DIV 拉去年分红汇总
- 实际扣税率是否 10%（W-8BEN 生效）
- 境外所得申报提醒（如适用）

### 8. 文件膨胀检查

- records/investment-ledger.md 是否超 500 行？→ 归档
- records/scan-history.md 是否超 500 行？→ 归档
- monthly-process.md 是否超 12 段？→ 归档
- archive/YYYY/INDEX.md 是否更新

## 硬规则

- 8 项全做，不跳
- 第 4 项预测回测必须数字化，不凭感觉
- 第 6 项持仓复核必须对照当初 thesis，不凭当前价格
- 发现问题 → 立即走提案流程（不等到季度扫描）
