# 季度自审清单

> 每季度末（3/31、6/30、9/30、12/31）必跑。
> 目的：发现规则是否被破、心态是否异常、规则是否过时。
> 产出 `research/quarterly-scans/YYYY-Qn/self-audit.md`

---

## 触发

- 每季度末日历自动触发
- 用户说"季度自审"

---

## 必答 10 题

### A. 规则遵守情况（Claude 自审）

**A1. 本季度生成过任何提案吗？如有，触发条件是否满足？**
- 查 `research/proposals/archive/` 最近 3 个月文件
- 每份提案对照触发条件
- 有无"讨好型提案"（触发未满足却生成）

**A2. 本季度有没有和自己之前立场矛盾的发言？**
- 扫描对话记录（git commit messages）
- 找"推翻前几天说的"场景
- 如有 → 记录何时何情况下发生

**A3. 本季度有没有破 hook 门禁？**
- 查 `.claude/.proposal-approved` 标记的使用记录
- 是否走过完整提案 → 复审 → 冷却（如有规则） → 批准流程？

### B. 行为纪律（用户审）

**B1. 本季度 3 个月 DCA 都按时执行了吗？**
- 查 `records/investment-ledger.md`
- 如有中断 → 原因是什么？是否写入 decision-journal？

**B2. 本季度用户有没有"情绪化追问"但被 Claude 正确拒绝？**
- 查 `records/decision-journal.md`
- 记录次数 + Claude 响应质量

**B3. 本季度有没有因情绪想动组合的冲动？**
- 即使没改 → 记录冲动本身
- 未来如果反复出现某类冲动 → IPS 可能需更新规则

### C. 数据一致性

**C1. ledger / holding-summary / performance 三处数据一致吗？**
- 当前持股 == Σ(ledger 累计)？
- performance 月末市值 == holding-summary 当前市值？
- 如不一致 → 以 ledger 为权威，重算其他

**C2. 实际 Benchmark 对比数据准确吗？**
- 本季度 SPY 实际回报 vs performance 记录
- 本季度 Policy 回报（70% SPY + 20% GLD + 10% BIL）
- 误差 > 0.5% 需重算

### D. 规则过时检查

**D1. portfolio-config.txt 里有没有规则本季度没用上的？**
- 如"周风控"一次没触发 → 规则合适
- 如"紧急触发器"一次没触发 → 正常（设计如此）
- 如"再平衡 ±5%"一次没触发但持仓确实偏了 > 5% → **规则失效**

**D2. IPS 第 3 节风控阈值还合适吗？**
- 当前组合实际波动率 / 最大回撤
- 如一直远低于 -30% → 可能风险承受可以更高（升级方案）
- 如接近 -30% → 设计合理

**D3. 有没有发现新的"应该固化"的规则？**
- 本季度遇到的任何"重复决策"
- 如每次都凭感觉判断某事 → 应该写进 runbook

### E. 宏观 + 市场状态

**E1. 当前宏观 regime 变化了吗？**
- 对比上季度 regime（concentration / broadening / 等）
- 如变化 → 评估组合倾斜是否需要调整
- 调整走提案流程

## 硬规则

- 10 题全答（不能跳）
- 每题必须有数据/文件引用（不能凭感觉）
- 发现规则失效 → 立即走 IPS/portfolio-config 更新提案
- 发现数据不一致 → 立即重算
