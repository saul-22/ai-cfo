# 边界场景与应急预案

> 15-20 年投资周期中可能遇到的所有非常规场景。
> 每个场景有明确的触发条件 + 处理流程 + 涉及文件。
> 最后更新：初始版本

---

## A. 持仓异常事件

### A1. 股票拆分（Stock Split）

**触发：** 持仓股票宣布拆分（如某股 10:1）
**影响：** 股数 ×10，股价 ÷10，总市值不变，但 ledger 和 holding-summary 的股数需更新
**处理：**
1. 在 `records/investment-ledger.md` 对应月份追加备注："<TICKER> 10:1 split on YYYY-MM-DD，股数 X→Y，成本均价 $A→$B"
2. 更新 `holding-summary.md` 的累计持股和成本均价
3. `performance.md` 不受影响（按市值算）
4. `portfolio-versions.md` 不变（比例不变）
5. git commit 记录

### A2. 持仓公司被收购（M&A）

**触发：** 持仓公司被收购（如某只 ETF 持仓中某公司被收购影响小，但单只持股被收购则影响大）
**影响：** 股票可能转换为现金 / 新公司股票 / 混合
**处理：**
1. 如果收到现金 → 记入 ledger，临时归入 T4 现金仓
2. 如果收到新公司股票 → 触发 stock-audit 评估新公司
3. 无论哪种 → 写 `research/proposals/remove-*.md` 或 `swap-*.md` 提案
4. 在 `portfolio-versions.md` 记录被动变更

### A3. ETF 清盘 / 重组

**触发：** 持仓 ETF 宣布清盘、合并、或重大指数变更
**影响：** 需要找替代 ETF
**处理：**
1. 触发紧急 stock-audit
2. 找替代（同类指数 ETF 或近似 ETF）
3. 写 `swap-*.md` 提案

### A4. 分红到账

**触发：** <券商>账户收到分红（持仓中分红型标的定期分红）
**影响：** 现金增加，需决定再投资还是留存
**处理：**
1. 在 `records/investment-ledger.md` 追加分红记录段：

```
### YYYY-MM 分红记录
| 代码 | 分红金额($) | 扣税率 | 实收($) | 处理 |
|------|-----------|--------|--------|------|
| <TICKER> | X.XX | 10% | X.XX | 再投资/留存 |
```

2. 检查扣税率是否 10%（W-8BEN 生效）。如果 30% → 立即重签 W-8BEN
3. **分红再投资决策：**
   - 单笔分红 < $10 → 留存（手续费不划算）
   - 单笔分红 ≥ $10 → 再投资到同一只股票（<券商> DRIP 设置）
   - 或累积到下月定投一起买
4. 更新 `performance.md`（分红计入总回报）

### A5. 退市预警

**触发：** 持仓股票收到退市通知或转 OTC
**影响：** 流动性丧失
**处理：**
1. 立即触发紧急 stock-audit
2. 评估是否在退市前卖出
3. 写 `remove-*.md` 提案

## B. 资金事件

### B1. 一次性大额入金（奖金、遗产、卖房等）

**触发：** 用户获得超出月度<月预算>的一次性资金
**处理：**
1. **不一次性全投**（择时风险）
2. 分 3-6 个月等额投入（DCA 原则不变）
3. 临时提高月投入金额，按当前版本比例分配
4. 在 `records/investment-ledger.md` 标注"额外入金"
5. 在 `records/decision-journal.md` 记录为什么选择分 N 个月

### B2. 月投入金额调整

**触发：** 按 IPS 阶段计划，递增年起从<月预算 A>→<月预算 B>
**时间点：**
- 递增年 1：<月预算 A>→<月预算 B>
- 递增年 2：<月预算 B>→<月预算 C>
- 或收入变化时提前/推迟

**处理：**
1. 写 `research/proposals/rule-budget-increase.md` 提案
2. 更新 `portfolio-config.txt` 的预算行
3. 更新 `ips.md` 的阶段表
4. buy-order.txt 自动按新预算生成
5. **比例不变**，只是每只的绝对金额增加

### B3. 提前提款

**触发：** 用户因紧急需要从组合提款
**处理（按 IPS 第 11 节优先级）：**
1. 先动 T4 现金仓
2. 其次赎回流动性最好的非核心仓（如黄金 ETF，非美居民资本利得免税）
3. 最后才动 T1/T2（核心仓）
4. 写 `research/proposals/phase-withdrawal.md` 提案
5. 在 `records/decision-journal.md` 记录
6. 提款后重新计算各仓位比例，评估是否需要再平衡

### B4. 月度执行异常（漏做 / 部分成交 / 延迟）

**触发：** 25 日定投未按计划完成
**处理：**

**情况 1：部分成交（如 5 只成交 1 只未成）**
1. 已成交的记入 ledger 实际字段
2. 未成交的在 ledger 备注段写"4/25 XXX 未成交，原因：[流动性/价格限制/其他]"
3. 下月 5/25 定投时**优先补足**该只（把 X 月缺的金额加到 Y 月同一只的配额）
4. buy-order.txt 下月生成时自动包含补买提示
5. decision-journal 简短记录

**情况 2：某只部分成交（如某只只买了 $150 而非 $175）**
1. 已成交的按实际记入
2. 缺额 ≤$50 → 记入 ledger 备注，下月补
3. 缺额 >$50 → 当月手动补足（当月内重新下单），如仍不行转到情况 1

**情况 3：整月未执行（忘做、出差、券商故障）**
1. 下月 DCA **不补偿性加投**（避免短期择时风险）
2. 在 decision-journal 记录"X 月未执行原因 + 为什么不补"
3. 汇总表对应月份写"未执行"
4. performance.md 月度明细标注"跳过月"（当月投入为 0，回报按市值变化算）
5. 如连续 2 月未执行 → 触发 IPS 审查（是否收入状态变化）

**情况 4：汇率突变（单日 >±2%）**
1. 4/25 开盘前检查汇率 vs 生成 buy-order 时的汇率
2. 变化 >2% → 用当日实际汇率重算 buy-order（美元金额不变、本币金额变）
3. 变化 >5% → 按 portfolio-config.txt 异常模式延后 1 周执行

**情况 5：参考价过期（4/25 价格 vs 4/22 参考价变化 >5%）**
1. 立即重算 buy-order（按 4/25 当日收盘价）
2. 估算股数相应调整
3. 不延后执行（DCA 不择时）

## C. 里程碑触发

### C1. 资产达 $60,000（遗产税警戒线）

**预估时间：** 视投入累计速度
**触发检测：** 每月更新 `holding-summary.md` 时自动检查总市值
**处理：**
1. 启动 UCITS ETF 替代研究（爱尔兰注册 ETF 替代美国注册 ETF）
2. 写 `research/proposals/phase-estate-tax.md` 提案
3. 咨询跨境税务师

### C2. 资产达 100 万元（进入成长期）

**触发检测：** `performance.md` 累计市值 ≥ 1,000,000
**处理：**
1. 写 `research/proposals/phase-growth.md` 提案
2. T3 加入 TLT（长期国债 ETF）
3. 更新 `portfolio-versions.md` 为 vX
4. 更新 `ips.md` 阶段

### C3. 资产达 300 万元（进入转型期）

**处理：**
1. 主动型 ETF→宽基 ETF（降波动）
2. 单股集中度降到 ≤ 10%
3. 完整 `phase-*.md` 提案

### C4. 资产达 500 万元（进入收获期）

**处理：**
1. 30% 固收配置
2. 启动 4% 年取款规则
3. 完整 `phase-*.md` 提案

## D. 券商 / 账户事件

### D1. <券商>出问题（系统故障、倒闭、政策变更）

**处理：**
1. SIPC 保险覆盖 $500,000（其中现金 $250,000）
2. 超出部分需转移到其他券商
3. 备选券商：Interactive Brokers（IBKR）、Charles Schwab
4. 转移流程：ACAT transfer（券商间直接转）

### D2. 开设第二个券商账户（规模 > 50 万时评估）

**触发：** 资产超 SIPC 保险 $500k / 工具需求（期权等）
**处理：** 届时根据实际情况决定账户分布方案。当前不急。

### D3. 券商持仓对账

**频率：** 每季度（Q3 扫描时顺便做）
**处理：**
1. 登录<券商>导出持仓
2. 对比 `holding-summary.md` 的股数
3. 如有差异（分红再投资、拆分、舍入误差）→ 以券商为准，更新 ledger

## E. 跨年事件

### E1. 年度归档（每年 12/31）

**处理：**
1. 当年 `records/investment-ledger.md` 复制到 `investment/archive/YYYY/ledger-YYYY.md`
2. 当年 `records/scan-history.md` 复制到 `investment/archive/YYYY/scan-history-YYYY.md`
3. 原文件清空历史段落，只保留最近 3 个月（避免文件膨胀）
4. `investment/archive/YYYY/INDEX.md` 更新年度汇总行
5. `performance.md` 追加年度汇总段
6. `records/decision-journal.md` 追加年度复盘段
7. 记忆季度审查（移过期记忆到 `memory/archive/`）
8. git commit + push

### E2. 年度 IPS 审查（每年 3 月）

**处理：**
1. 读 `ips.md` 全文
2. 检查每项是否仍准确（收入、风险容忍、流动性需求、税务状态）
3. 如有变化 → 写 `rule-*.md` 提案
4. W-8BEN 检查（3 年有效期）
5. 1099-DIV 税务复盘

### E3. 月投入调整年份（2031、2036）

见 B2。

## F. 技术 / 工具事件

### F1. FMP API key 过期或被 revoke

**处理：**
1. 去 FMP 网站重新生成
2. 更新 `.env.local`
3. 验证可用

### F2. GitHub 仓库访问丢失

**处理：**
1. 本地 git 仍完整（所有历史都在）
2. 重新创建 GitHub 仓库 + push
3. 或换其他 git 托管（GitLab、Bitbucket）

### F3. 第三方 skill 仓库删除或大改

**处理：**
1. 我们只保留 SKILL.md 方法论文件（已在本地 git）
2. 原仓库删了不影响我们
3. 大改时评估是否同步更新

### F4. Claude 版本升级导致行为变化

**处理：**
1. CLAUDE.md 是强制规则，新版本仍会读取
2. 如果新版本不遵守某些规则 → 用户反馈 → 更新 CLAUDE.md 措辞
3. 核心数据在文件里，不依赖 Claude 记忆

## G. 人生事件

### G1. 结婚

**影响：** 可能需要调整流动性需求、增加投入、或开联名账户
**处理：** 触发 IPS 全盘审查

### G2. 生育

**影响：** 支出增加、可能需要教育基金
**处理：** 触发 IPS 审查，可能降低月投入或增加流动性需求

### G3. 移民 / 税务身份变更

**影响：** 如果变成美国税务居民 → 资本利得不再免税、遗产税规则变化
**处理：** 立即触发 IPS + tax-compliance 全面审查

### G4. 身故

**处理：**
1. `README.md` 是入口（家人能看懂）
2. `ips.md` 第 12 节有传承计划
3. <券商>有受益人设置（TOD）
4. 如资产 > $60k → UCITS ETF 替代方案应已执行（C1）

## 触发检测机制

**以下检查嵌入现有 SOP，不需要额外流程：**

| 检查项 | 嵌入时机 | 检测方法 |
|---|---|---|
| 股票拆分 | 每月更新 holding-summary 时 | 股价突变 >50% 但市值不变 |
| 分红到账 | 用户反馈成交时顺便问 | "本月有分红吗？" |
| 里程碑（$60k/100万/300万/500万）| 每月更新 performance.md 时 | 自动对比阈值 |
| 券商对账 | 每季度扫描时 | 对比 holding-summary vs 券商导出 |
| 年度归档 | 12 月定投闭环后 | 日历触发 |
| IPS 年审 | 3 月定投闭环后 | 日历触发 |
| 月投入调整 | 每年 1 月检查 | 对比 IPS 阶段计划 |
| W-8BEN 到期 | 每年 3 月 | 日历触发（3 年有效期，签署日 + 3 年到期需续签）|
