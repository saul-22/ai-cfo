# Runbook B：当前组合审查（Current Portfolio Review）

> 只看当前生效组合是否仍合规、是否需要调整。
> 季度扫描日、紧急事件、里程碑触发时走这个。
> 产出**能够执行**（通过提案流程）。

---

## 触发

- 季度扫描日（8/1、11/1、2/1）
- 年度再平衡（3/25）
- 紧急触发器激活（单股 -40%、SEC、CEO 离职、SPY -20%）
- 里程碑达到（100万/300万/500万）
- 偏离违规（±5% 连续两月 / ±10%）

## 绝对禁止

- ❌ 白纸构建
- ❌ "如果从零开始"式思考
- ❌ 无触发条件时走此流程

## 流程（5 步）

### Step 1：触发条件确认
记录是哪个触发条件激活。

### Step 2：当前组合健康检查
- 每只持仓重新过 8 filter
- 对比建仓时 thesis（scan-history / decision-journal / portfolio-versions）
- 标记：仍合规 / 边缘 / 红灯

### Step 3：IPS 约束对照
- 年化、回撤、5 年滚动、DCA 各项是否仍满足

### Step 4：调整建议
只做必要变动：
- 红灯股 → 替换候选（必须过 8 filter 且 ≥ 6/8）
- 边缘股 → 降仓位或标记下次审查
- 合规股 → 保持

### Step 5：提案 or 无需变更
- 有变更 → 写 `research/proposals/swap-/reweight-*.md`
- 无变更 → 写 `research/quarterly-scans/YYYY-Qn/hold-decision.md`

## 输出格式

```markdown

## Step 1：触发确认
[哪个触发条件]

## Step 2：持仓健康检查
| 代码 | 权重 | 8 filter 重测 | vs 建仓 thesis | 判定 |
|---|---|---|---|---|
| <TICKER> | <X>% | 7/8 | 符合 | 仍合规 |
| <TICKER> | <X>% | 5/8 | 符合 | 边缘 |
| ... | | | | |

## Step 3：IPS 合规
[每项约束验证]

## Step 4：调整建议
[HOLD / 调整列表]

## Step 5：产出
[提案路径 或 hold-decision 路径]
```

**有 "vs 建仓 thesis" 段落。有 "执行建议"。围绕现有组合。**

---

## 用户用法

- 季度扫描 → 这个流程告诉你当前组合还行不行
- 紧急事件 → 这个流程告诉你要不要换某只
- 直接决定执行（通过提案）
